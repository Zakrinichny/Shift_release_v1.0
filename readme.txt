Програмама призначена для створення робочої зміни у ВРСЗ
Програма запускається подвійним кліком по архіву.
Програма створена в Eclipse (Version: 2019-09 R (4.13.0)).
Для роботи програми необхідно мати актуальну версію JRE на комп'ютері (для запуску архіву, скачати можна тут
https://www.oracle.com/java/technologies/javase-jre8-downloads.html) та наступні файли в одному каталогу з архівом:
src/
	data/
		blocked.txt
		data.txt
		information.txt
		pattern.txt
	icon/
		add.png
		addshift.png
		delshift.png
		exit.png
		info.png
		lock.png
		min.png
		pl.png
		rem.png
		save.png
		unlock.png
Функціонал:
- додавання співробітників (із зазначенням постів, на які можуть заступати співробітники, співробітник повинен заступати, як мінімум, на один пост, мати ім’я, як мінімум, з одної букви, інакше такого співробітника зберегти неможливо)
- видалення співробітників 
- змінити співробітника неможливо, якщо ви помилилися в імені співробітника або зазначили невірні пости, або для співробітника треба додати (змінити) пости,  треба видалити такого співробітника та створити його заново
- у списку постів не зазначен пост №691, тому що на нього завжди заступає начальник зміни, програма автоматично заповнює цей пост у таблиці збереженного файлу
- створення довільної кількісті змін в одному файлі
- додавання довільної кількісті співробітників на один пост
- програма автоматично пропонує дату для зміни (через день від сьогоднішньої, але і є можливість вибрати довільну дату від сьогоднішньої до 10 днів наперед. Дату краще обирати одразу перш ніж обирати співробітників на пости, при змінні дати усі пости оновлюються, вибрані Вами співробітники зіб’ються)
- программа автоматично пропонує назву для файлу, залежно від кількості змін та дат, на які ці зміни створено
- программа зберігає результат у файлі формату Word (незалежно від кількості даданих змін, усі вони будуть збережені в одному файлі, для створення змін у різних файлах зберігайте їх окремо, тобто створили зміну – зберегли, створили іншу – зберегли…)
- для зручності роботи з  декількома змінами є можливість зміни виду на горизонтальний чи вертикальний (обраний за замовчуванням)
- створювання шаблонів змін (обравши співробітників для кожного поста, їх можна зберегти у шаблон, це спрощює створення зміни, аби не додавати кожного співробітника окремо кожного разу. Для збереження шаблону оберіть для кожного поста співробітника та збережіть шаблон, оберіть назву для шаблону)
- загрузка шаблонів (збереженні попередньо шаблони можна загружати за їх назвою. Якщо в шаблоні присутній заблокований співробітник програма попередить про це, та прибире його з відповідного поста, такий пост залишиться пустим! Увага: якщо на головній панелі програми відображено декілька змін, шаблон загрузиться на останню. Рекомендований порядок роботи з декількома змінами: обрати дату, загрузити шаблон, дадати другу зміну, обрати дату, загрузити для неї шаблон, додати треттю зміну і так далі. Якщо у Вас створено декілька змін, і Ви забули загрузити шаблон на першу, Ви зможете загрузити шаблон тільки на останню зміну, або обирайте кожного співробітника для кожного поста окремо для першої зміни, або видаляйте зміни до першої, загружайте для неї шаблон і вже потім додавайте наступні зміни згідно рекомендованного порядку роботи з декількома змінами)
- видалення шаблонів
- шаблон змінити неможливо, для цього видаліть старий шаблон, та створіть новий потрібний Вам шаблон
- блокування співробітника (у зв’язку з хворобою, відрядженням, відпусткою та ін. Заблокований співробітник видаляється зі списку поста, на який він може заступати до дати (включно), яка була вказана при його блокуванні. По закінченню дати, співробітник автоматично розблоковується. Заблокувати співробітника можливо, як мінімум, на наступнку дату, тобто на завтра, сьогоднішньою датою заблокувати співробітрика неможливо!)
- розблоковування співробітника завчасно (у разі потреби розблокувати його раніше дати, вказанної при блокуванні)
- переглянути дату блокування співробітника можна в подпункті меню «Розблокувати співробітника», обравши відповідного співробітника зі списку
- до кожної операції є доступ через відповідні кнопки, пункти меню або функціональні клавіші (вказані в пунктах меню)
Результат роботи програми:
файл .docx з заповненою таблицею (або декількома таблицями) робочої зміни на певну дату.
